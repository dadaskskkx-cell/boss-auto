# 给同事使用说明

## 这是什么

这是一个只支持 `Boss 直聘 PC 客户端` 的 Mac 脚本包。

作用是：

- 启动后先输入本次岗位名称和岗位 JD
- 程序自动根据 JD 生成关键词和硬筛规则
- 读取 Boss 推荐列表里的候选人卡片
- 对符合规则的候选人自动点击 `打招呼`
- 自动下滑继续扫描更多候选人

不支持网页端。

## 同事需要准备什么

同事机器上只需要这 3 样：

1. `Boss 直聘 PC 客户端`
2. `Python 3.12` 或 `Python 3.13`
3. 这个文件夹：`boss-auto-script-mac`

## 模型 OCR 要不要另外配

这份分发包里包含的是 `config/config.yaml` 模板，不会包含别人机器上的真实 key。

同事第一次拿到包后，需要先把自己的模型配置填进去，至少要改：

- `llm.api_key`
- `llm.vision_api_key`

默认模板已经按 `GLM` 接口写好，正常只需要把 key 替换掉。

## 同事怎么安装 Python

官方下载页：

`https://www.python.org/downloads/macos/`

推荐安装：

- `Python 3.13.x`
- 或 `Python 3.12.x`

不要选预发布版本。

安装完成后，可以在终端验证：

```bash
python3 -V
```

## 同事怎么运行

1. 打开 `Boss 直聘 PC 客户端`
2. 登录账号
3. 打开 `boss-auto-script-mac`
4. 双击 `launch.command`
5. 如果还没配过 key，程序会先自动生成 `config/config.yaml` 模板，并提示你先去填写
6. 填完 key 后，再次双击 `launch.command`
7. 第一次运行会自动安装依赖，耐心等一下
8. 按提示输入岗位名称
9. 粘贴本次岗位 JD
10. 输入完成后，单独一行输入：

```text
/done
```

11. 检查程序自动生成的规则
12. 直接回车使用默认值，或手动改规则
13. 确认后，脚本开始执行

## 运行时注意

- Boss 客户端要保持打开
- 推荐列表页面要处于可操作状态
- 脚本运行时尽量不要频繁切走 Boss 窗口
- 如果要强制停止，把鼠标移到屏幕左上角

## 遇到问题先看什么

如果启动失败，优先检查：

1. 有没有安装 `Python 3.12/3.13`
2. `config/config.yaml` 里的 key 有没有填写
3. Boss 客户端是否已经登录
4. 当前是不是在 Boss 客户端，而不是网页端
5. 网络是否能正常安装依赖和访问模型接口

## 这份包里最重要的文件

- `launch.command`
  启动入口
- `config/config.yaml`
  模型和运行配置模板/实际配置
- `docs/同事试用说明.md`
  简版说明
- `docs/给同事使用说明.md`
  发同事看的完整版说明
